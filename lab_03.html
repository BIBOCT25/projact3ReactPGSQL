Теория

🪝 Хуки React: useState и useEffect

Хук (Hook) — это специальная функция React, которая добавляет возможности компоненту. Имена всегда начинаются с use. Два самых важных хука — useState и useEffect.

useState — реактивная переменная

Обычная переменная в JS не перерисовывает экран при изменении. useState — перерисовывает.

Обычная переменнаяuseState
let count = 0
count = count + 1
// экран не обновится		 const [count, setCount] = useState(0)
setCount(count + 1)
// экран обновится автоматически		 
// Синтаксис useState:
const [значение, функция_изменения] = useState(начальное_значение)

// Примеры:
const [count,    setCount]    = useState(0)          // число
const [loading,  setLoading]  = useState(true)        // булево
const [products, setProducts] = useState([])          // массив
const [error,    setError]    = useState(null)         // null или строка

useEffect — побочные эффекты

Используется для кода, который должен выполниться после того, как компонент появился на экране: загрузка данных, подписки на события, таймеры.

// Синтаксис useEffect:
useEffect(() => {
  // этот код запустится после рендера
}, [зависимости])

// [] — пустой массив = запустить ОДИН раз при монтировании компонента
// [id] — запускать при каждом изменении переменной id
// без [] — запускать при КАЖДОМ рендере (опасно!)

Шаг 1

⚙️ Настроить proxy для fetch

1

Добавить proxy в vite.config.ts

Без proxy React (порт 5173) не может обращаться к Express (порт 3001) из-за ограничений браузера (CORS). Vite умеет проксировать запросы — перенаправлять /api/* на наш сервер.

frontend/vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3001', // адрес нашего Express-сервера
        changeOrigin: true,             // менять заголовок Origin
      }
    }
  }
})
💡 Теперь вместо fetch('http://localhost:3001/api/products')
можно писать просто fetch('/api/products') — короче и удобнее.
Шаг 2

🌐 Создаём API-клиент

2

Вынести fetch-запросы в отдельный файл

Хорошая практика — держать все обращения к серверу в одном месте, а не разбрасывать fetch по компонентам.

frontend/src/api/products.ts
// Тип товара — описываем форму данных которые приходят с сервера
export interface Product {
  id:       number
  name:     string
  price:    number
  oldPrice: number | null  // может быть null если скидки нет
  stock:    number
}

// Получить все товары
export async function fetchProducts(): Promise<Product[]> {
  const res = await fetch('/api/products')
  if (!res.ok) throw new Error('Ошибка загрузки товаров') // бросаем ошибку если 4xx/5xx
  return res.json()
}

// Добавить товар в корзину
export async function addToCart(productId: number): Promise<void> {
  const res = await fetch('/api/cart', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({ productId })
    // JSON.stringify превращает объект { productId: 1 }
    // в строку '{"productId":1}' для отправки
  })
  if (!res.ok) throw new Error('Не удалось добавить в корзину')
}
Шаг 3

🃏 Компонент ProductCard

3

Создать карточку товара

Компонент получает данные через props (параметры) и рисует HTML. Кнопка вызывает callback переданный родителем.

frontend/src/components/ProductCard.tsx
import { useState } from 'react'
import { Product, addToCart } from '../api/products'

// Описываем props — что компонент принимает снаружи
interface Props {
  product:      Product        // данные товара
  onCartChange: () => void    // функция которую вызовем после добавления
}

export function ProductCard({ product, onCartChange }: Props) {
  // Локальное состояние — идёт ли добавление прямо сейчас
  const [adding, setAdding] = useState(false)
  const [added,  setAdded]  = useState(false)

  const handleAdd = async () => {
    setAdding(true)          // показать спиннер
    try {
      await addToCart(product.id)
      setAdded(true)          // показать галочку
      onCartChange()          // сообщить родителю — пусть обновит счётчик
      setTimeout(() => setAdded(false), 2000) // через 2с вернуть обычный вид
    } finally {
      setAdding(false)
    }
  }

  // Форматирование цены: 4990 → "4 990 ₽"
  const fmt = (n: number) => n.toLocaleString('ru-RU') + ' ₽'

  return (
    <div className="product-card">
      {'/* Картинка-заглушка — в реальном проекте будет img */'}
      <div className="card-img">🛍️</div>

      <div className="card-body">
        <h3 className="card-name">{product.name}</h3>

        <div className="card-price">
          <span className="price-current">{fmt(product.price)}</span>
          {product.oldPrice && (  // рендерить только если oldPrice не null
            <span className="price-old">{fmt(product.oldPrice)}</span>
          )}
        </div>

        <button
          className=`btn-cart ${added ? 'btn-added' : ''}`
          onClick={handleAdd}
          disabled={adding || product.stock === 0}
        >
          {adding ? 'Добавляем…' : added ? '✓ Добавлено' : 'В корзину'}
          {'/* тернарный оператор: если adding → "Добавляем…" */'}
        </button>
      </div>
    </div>
  )
}
Шаг 4

📋 Компонент каталога — загрузка данных

4

Создать страницу каталога

Это главный компонент. Он загружает товары с сервера и передаёт их в ProductCard.

frontend/src/pages/CatalogPage.tsx
import { useState, useEffect } from 'react'
import { Product, fetchProducts } from '../api/products'
import { ProductCard } from '../components/ProductCard'

export function CatalogPage() {
  // Три состояния загрузки данных с сервера:
  const [products, setProducts] = useState<Product[]>([]) // данные
  const [loading,  setLoading]  = useState(true)           // идёт ли загрузка
  const [error,    setError]    = useState<string | null>(null) // текст ошибки
  const [cartCount, setCartCount] = useState(0)              // кол-во товаров в корзине

  // useEffect с [] — загрузить данные один раз при открытии страницы
  useEffect(() => {
    fetchProducts()
      .then(data => {
        setProducts(data)  // сохранить товары в состояние
        setLoading(false)  // скрыть экран загрузки
      })
      .catch(err => {
        setError(err.message) // показать ошибку
        setLoading(false)
      })
  }, []) // [] = запустить один раз

  // Вызывается когда пользователь добавил товар в корзину
  const handleCartChange = () => {
    setCartCount(prev => prev + 1) // prev — предыдущее значение состояния
  }

  // ── РЕНДЕР ──

  // Если загружается — показать skeleton (анимированные заглушки)
  if (loading) return (
    <div className="catalog-grid">
      {[1,2,3].map(i => <div key={i} className="skeleton-card" />)}
    </div>
  )

  // Если ошибка — показать сообщение
  if (error) return (
    <div className="error-box">
      ⚠️ {error}. Убедитесь что сервер запущен.
    </div>
  )

  return (
    <div>
      <header className="page-header">
        <h1>Каталог</h1>
        <span className="cart-badge">🛒 {cartCount}</span>
      </header>

      <div className="catalog-grid">
        {products.map(product => (
          {'/* .map() — обход массива, для каждого товара рисуем карточку */'}
          <ProductCard
            key={product.id}          // key — уникальный ключ для React
            product={product}         // передаём товар как prop
            onCartChange={handleCartChange}
          />
        ))}
      </div>
    </div>
  )
}
