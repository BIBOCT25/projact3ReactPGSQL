Теория

🧠 Что такое стек и зачем три части?

Стек — это набор технологий, которые работают вместе. Представьте ресторан: есть зал (то, что видит гость), кухня (логика), и склад (данные). В нашем случае: React = зал, Node.js/Express = кухня, PostgreSQL = склад.

⚛️ React

Работает в браузере. Рисует кнопки, карточки, корзину. Когда нажимаете кнопку — React обновляет экран мгновенно, без перезагрузки страницы.

🖥️ Node.js + Express

Работает на сервере. Принимает запросы от React, проверяет права, читает/пишет данные в базу. Это посредник между интерфейсом и данными.

🐘 PostgreSQL

Хранит данные навсегда. Пользователи, товары, заказы, корзина — всё здесь. Данные не пропадают при перезагрузке страницы или сервера.

❓ Зачем не обойтись одним React?
React живёт только в браузере. Закрыли вкладку — всё исчезло. Чтобы корзина сохранялась между сессиями, пользователи могли войти в аккаунт, а заказы не пропадали — нужна база данных на сервере.
Архитектура

🏗 Как они общаются?

┌─────────────────────────────────────────────┐ │ БРАУЗЕР (React) │ │ Компоненты → useState → fetch('/api/...') │ └───────────────────┬─────────────────────────┘ │ HTTP запросы: GET, POST, PATCH, DELETE ▼ ┌─────────────────────────────────────────────┐ │ СЕРВЕР (Node.js + Express) │ │ Роуты → проверка JWT → SQL-запрос │ └───────────────────┬─────────────────────────┘ │ SQL: SELECT, INSERT, UPDATE, DELETE ▼ ┌─────────────────────────────────────────────┐ │ PostgreSQL │ │ Таблицы: users, products, cart_items... │ └─────────────────────────────────────────────┘

Пример: нажали «Добавить в корзину»

#КтоЧто делает
1ReactПользователь нажал кнопку
2ReactОтправляет POST /api/cart с { product_id: 42 }
3ExpressПолучил запрос, проверил токен пользователя
4ExpressВыполняет INSERT INTO cart_items ...
5PostgreSQLСохранил строку в таблицу
6ExpressВернул { success: true, total: 1490 }
7ReactОбновил счётчик корзины, показал уведомление

💡 Что такое ORM?
ORM — это прослойка между кодом и базой данных. Вместо SQL-запроса
SELECT * FROM users WHERE id = 1
вы пишете TypeScript-код:
prisma.user.findUnique({ where: { id: 1 } })
Читается понятнее, TypeScript проверяет правильность полей.
Практика

🚀 Устанавливаем всё с нуля

1

Установить Node.js

Скачайте LTS-версию с официального сайта. После установки проверьте в терминале:

# Открыть терминал (cmd / PowerShell / Terminal) и проверить:
node --version    # должно вывести v22.x.x
npm --version     # должно вывести 10.x.x
📥 Где скачать: https://nodejs.org → кнопка «LTS»
2

Установить PostgreSQL

Скачайте и установите PostgreSQL 16. В процессе установки запомните пароль для пользователя postgres.

# Windows: скачать с https://www.postgresql.org/download/windows/
# Запустить установщик, запомнить пароль superuser

# После установки — создать базу данных:
# Открыть pgAdmin (устанавливается вместе) или psql в терминале

psql -U postgres
-- Введите пароль, который задали при установке

CREATE DATABASE shop_db;
CREATE USER shop_user WITH PASSWORD 'mypassword123';
GRANT ALL PRIVILEGES ON DATABASE shop_db TO shop_user;
\q
3

Создать структуру проекта

Наш проект будет состоять из двух папок: фронтенд и бэкенд.

# Создать корневую папку проекта
mkdir my-shop
cd my-shop

# Создать фронтенд (React + Vite)
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
cd ..

# Создать бэкенд (Node.js + Express)
mkdir backend
cd backend
npm init -y
4

Установить зависимости бэкенда

Находясь в папке backend/, установите нужные пакеты:

# Основные пакеты
npm install express cors dotenv

# Prisma (ORM для PostgreSQL)
npm install prisma @prisma/client

# TypeScript и инструменты разработки
npm install -D typescript ts-node nodemon @types/node @types/express

# Проверить — в node_modules появятся все папки
💡 Разница -D и без флага:
Без -D — нужно в продакшне (express, prisma)
С -D — только для разработки (typescript, nodemon)
5

Итоговая структура папок

После всех шагов должна получиться такая структура:

my-shop/
├── frontend/                 ← React + Vite (фронтенд)
│   ├── src/
│   │   ├── App.tsx
│   │   └── main.tsx
│   ├── package.json
│   └── vite.config.ts
│
└── backend/                  ← Node.js + Express (бэкенд)
    ├── src/
    │   └── index.ts          ← точка входа сервера
    ├── prisma/
    │   └── schema.prisma     ← описание таблиц БД
    ├── .env                  ← настройки (пароль БД, секреты)
    └── package.json
Запуск

▶ Как запустить проект

Нужно открыть два терминала одновременно:

Терминал 1 — Бэкенд

cd my-shop/backend
npx nodemon src/index.ts

# Запущен на http://localhost:3001

Терминал 2 — Фронтенд

cd my-shop/frontend
npm run dev

# Открыть http://localhost:5173
⚠️ Частая ошибка: Если React запрашивает localhost:3001, а бэкенд не запущен — в консоли браузера появится ошибка Failed to fetch. Убедитесь, что оба терминала работают.
