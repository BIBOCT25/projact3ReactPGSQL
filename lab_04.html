Теория

🪝 Кастомный хук — зачем выносить логику

Кастомный хук — это обычная функция, которая:
  • начинается с use
  • использует другие хуки (useState, useEffect, ...)
  • возвращает данные и функции для компонента

Это способ вынести логику из компонента и переиспользовать её в нескольких местах.
Без хукаС хуком useCart
Весь useState, useEffect, fetch — прямо в компоненте корзины. 100+ строк логики внутри JSX. Компонент вызывает const { cart, updateQty, removeItem } = useCart() и занимается только отображением.

Пример простого кастомного хука

// Хук — просто функция с use-prefix
function useCounter(initial = 0) {
  const [count, setCount] = useState(initial)
  const increment = () => setCount(c => c + 1)
  const decrement = () => setCount(c => c - 1)
  const reset     = () => setCount(initial)
  return { count, increment, decrement, reset }
}

// Использование в компоненте:
function Counter() {
  const { count, increment, decrement } = useCounter(0)
  return (
    <div>
      <button onClick={decrement}>−</button>
      {count}
      <button onClick={increment}>+</button>
    </div>
  )
}

frontend/src/api/cart.ts
// Тип одной позиции корзины (приходит с сервера)
export interface CartItem {
  id:        number
  qty:       number
  productId: number
  userId:    number
  product: {            // вложенный объект из JOIN
    id:    number
    name:  string
    price: number
  }
}

// Ответ GET /api/cart
export interface CartResponse {
  items:     CartItem[]
  total:     number
  remaining: number     // сколько ₽ до бесплатной доставки
}

// Получить корзину
export async function fetchCart(): Promise<CartResponse> {
  const res = await fetch('/api/cart')
  if (!res.ok) throw new Error('Ошибка загрузки корзины')
  return res.json()
}

// Изменить количество (PATCH /api/cart/:id)
export async function updateCartItem(id: number, qty: number): Promise<void> {
  await fetch(`/api/cart/${id}`, {
    method:  'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({ qty })
  })
}

// Удалить позицию (DELETE /api/cart/:id)
export async function removeCartItem(id: number): Promise<void> {
  await fetch(`/api/cart/${id}`, { method: 'DELETE' })
}
Шаг 2

🪝 Хук useCart

2

Создать хук hooks/useCart.ts

Хук инкапсулирует всю логику: загрузку, обновление, удаление и оптимистичное обновление экрана.

frontend/src/hooks/useCart.ts
import { useState, useEffect, useCallback } from 'react'
import { CartResponse, fetchCart, updateCartItem, removeCartItem } from '../api/cart'

export function useCart() {
  const [cart,    setCart]    = useState<CartResponse | null>(null)
  const [loading, setLoading] = useState(true)

  // useCallback — мемоизация функции, чтобы не создавать заново при каждом рендере
  const reload = useCallback(async () => {
    try {
      const data = await fetchCart()
      setCart(data)
    } finally {
      setLoading(false)
    }
  }, [])

  // Загрузить корзину при монтировании
  useEffect(() => { reload() }, [reload])

  // Изменить количество — оптимистично
  const updateQty = async (itemId: number, newQty: number) => {
    if (newQty < 1) { return removeItem(itemId) } // qty < 1 → удалить

    // Optimistic update: сразу меняем UI, не ждём сервер
    setCart(prev => {
      if (!prev) return prev
      const items = prev.items.map(i =>
        i.id === itemId ? { ...i, qty: newQty } : i  // обновляем нужный элемент
      )
      const total = calcTotal(items)           // пересчитываем сумму
      return { ...prev, items, total, remaining: calcRemaining(total) }
    })

    await updateCartItem(itemId, newQty)  // отправляем на сервер в фоне
  }

  // Удалить позицию
  const removeItem = async (itemId: number) => {
    // Сразу убираем из UI
    setCart(prev => {
      if (!prev) return prev
      const items = prev.items.filter(i => i.id !== itemId) // filter = убрать элемент
      const total = calcTotal(items)
      return { ...prev, items, total, remaining: calcRemaining(total) }
    })
    await removeCartItem(itemId)
  }

  return { cart, loading, updateQty, removeItem, reload }
}

// Вспомогательные функции — считаем локально чтобы не ждать сервер
const FREE_DELIVERY = 3000  // порог бесплатной доставки в ₽

function calcTotal(items: any[]) {
  return items.reduce((sum, i) => sum + i.product.price * i.qty, 0)
  // reduce: начать с 0, для каждого элемента добавить цена × количество
}

function calcRemaining(total: number) {
  return Math.max(0, FREE_DELIVERY - total)
  // не может быть отрицательным — Math.max(0, ...)
}
💡 Optimistic Update — «оптимистичное обновление».
Мы сразу меняем UI как будто сервер уже ответил. Пользователь видит мгновенный отклик. Если сервер вернёт ошибку — откатим изменение. Так работают все крупные приложения: Twitter, Telegram, Notion.

frontend/src/pages/CartPage.tsx
import { useCart } from '../hooks/useCart'

export function CartPage() {
  const { cart, loading, updateQty, removeItem } = useCart()
  // Весь useState и fetch — внутри хука. Здесь только отображение.

  if (loading) return <p>Загрузка корзины…</p>
  if (!cart || cart.items.length === 0) return <p>Корзина пуста</p>

  const fmt = (n: number) => n.toLocaleString('ru-RU') + ' ₽'

  return (
    <div className="cart-page">
      <h1>Корзина</h1>

      {'/* Полоса бесплатной доставки */'}
      <FreeDeliveryBar remaining={cart.remaining} threshold={3000} />

      {'/* Список позиций */'}
      <div className="cart-items">
        {cart.items.map(item => (
          <div key={item.id} className="cart-item">

            <div className="item-name">{item.product.name}</div>

            {'/* Кнопки +/− */'}
            <div className="qty-control">
              <button onClick={() => updateQty(item.id, item.qty - 1)}>−</button>
              <span>{item.qty}</span>
              <button onClick={() => updateQty(item.id, item.qty + 1)}>+</button>
            </div>

            <span className="item-price">
              {fmt(item.product.price * item.qty)}
            </span>

            <button
              className="btn-remove"
              onClick={() => removeItem(item.id)}
            >🗑</button>

          </div>
        ))}
      </div>

      {'/* Итоговая сумма */'}
      <div className="cart-total">
        <span>Итого:</span>
        <strong>{fmt(cart.total)}</strong>
      </div>

      <button className="btn-checkout">Оформить заказ</button>
    </div>
  )
}
Шаг 4

📊 Компонент «До бесплатной доставки»

4

Создать FreeDeliveryBar.tsx

Компонент отображает прогресс-бар и текст. Получает только два числа — больше ничего не знает о корзине.

frontend/src/components/FreeDeliveryBar.tsx
interface Props {
  remaining: number   // сколько ₽ осталось
  threshold: number   // порог бесплатной доставки (3000)
}

export function FreeDeliveryBar({ remaining, threshold }: Props) {
  // Если remaining = 0 — доставка бесплатна
  if (remaining === 0) {
    return (
      <div className="delivery-bar delivery-free">
        🎉 Бесплатная доставка!
      </div>
    )
  }

  // Процент заполнения полосы
  const filled = remaining === 0
    ? 100
    : ((1 - remaining / threshold) * 100).toFixed(1)
  // Пример: remaining=750, threshold=3000 → (1 - 750/3000)*100 = 75%

  const fmt = (n: number) => n.toLocaleString('ru-RU')

  return (
    <div className="delivery-bar">
      <p>
        Ещё <strong>{fmt(remaining)} ₽</strong> до бесплатной доставки
      </p>
      <div className="bar-track">
        <div
          className="bar-fill"
          style={{ width: `${filled}%` }}  // ширина = % заполнения
        />
      </div>
    </div>
  )
}

import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'
import { CatalogPage } from './pages/CatalogPage'
import { CartPage }    from './pages/CartPage'
import './index.css'

function App() {
  return (
    <BrowserRouter>               // оборачиваем всё приложение
      <nav className="app-nav">
        <Link to="/">Каталог</Link>   // Link — вместо <a>, без перезагрузки страницы
        <Link to="/cart">🛒 Корзина</Link>
      </nav>

      <Routes>
        <Route path="/"      element={<CatalogPage />} />
        <Route path="/cart" element={<CartPage />}    />
      </Routes>
    </BrowserRouter>
  )
}

export default App
💡 Link vs <a>
Обычный тег <a href="/cart"> перезагружает страницу — React-состояние теряется. <Link to="/cart"> меняет URL без перезагрузки — приложение остаётся «живым».

CartPage вызывает useCart() → хук держит cart в useState → при updateQty(id, qty) → сначала меняем setCart (мгновенно), потом PATCH /api/cart/:id (в фоне) → экран не «мигает», пользователь не ждёт.
