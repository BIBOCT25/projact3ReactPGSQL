Теория

🔌 Что такое REST API и роуты

REST API — это набор URL-адресов (роутов), по которым React обращается к серверу. Каждый роут = адрес + HTTP-метод.
МетодURLЧто делаетАналогия
GET/api/productsПолучить список товаровПрочитать меню
GET/api/products/5Получить товар с id=5Выбрать одно блюдо
POST/api/cartДобавить товар в корзинуСделать заказ
PATCH/api/cart/3Изменить кол-во в позиции корзиныИзменить заказ
DELETE/api/cart/3Удалить позицию из корзиныОтменить блюдо
💡 Как это работает:
React делает fetch('http://localhost:3001/api/products') → Express получает GET-запрос к /api/products → Prisma выполняет SELECT * FROM products → Express отвечает JSON-массивом → React получает данные и рисует карточки товаров.
Шаг 1

📐 Описываем таблицы в Prisma

1

Инициализировать Prisma

В папке backend/ выполнить команду:

npx prisma init
# Команда создаст два файла:
# prisma/schema.prisma  — описание таблиц
# .env                  — переменные окружения (пароль БД)
2

Настроить файл .env

Откройте файл .env и замените строку с DATABASE_URL на свои данные:

.env
# Формат: postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@ХОСТ:ПОРТ/БАЗА
DATABASE_URL="postgresql://shop_user:mypassword123@localhost:5432/shop_db"
PORT=3001
⚠️ Файл .env — секретный!
Никогда не загружайте .env на GitHub. Добавьте в .gitignore строку .env
3

Описать таблицы в schema.prisma

Каждая модель Prisma = одна таблица в PostgreSQL. Откройте prisma/schema.prisma:

prisma/schema.prisma
// Настройки генератора — генерировать TypeScript-клиент
generator client {
  provider = "prisma-client-js"
}

// Настройки подключения к базе данных
datasource db {
  provider = "postgresql"     // тип базы — PostgreSQL
  url      = env("DATABASE_URL") // URL берём из .env
}

// Модель = таблица products
model Product {
  id       Int     @id @default(autoincrement()) // первичный ключ, автоинкремент
  name     String                                // название товара
  price    Float                                 // цена
  oldPrice Float?                                // старая цена (? = необязательное поле)
  stock    Int     @default(0)                   // количество на складе

  cartItems CartItem[] // связь: у товара может быть много позиций в корзинах
}

// Модель = таблица cart_items
model CartItem {
  id        Int @id @default(autoincrement())
  qty       Int @default(1) // количество единиц

  productId Int            // ID товара — внешний ключ
  userId    Int            // ID пользователя (добавим пользователя позже)

  product   Product @relation(fields: [productId], references: [id])
  // ↑ связь: поле productId ссылается на поле id в таблице Product

  @@unique([userId, productId]) // один пользователь не может добавить один товар дважды
}
Что значит @@unique([userId, productId])? Это составной уникальный индекс. Пользователь с id=1 может иметь товар с id=5 в корзине только один раз. Если хочет добавить ещё — нужно увеличить qty, а не создавать новую строку.
4

Применить схему к базе данных (миграция)

Миграция — это создание таблиц в PostgreSQL на основе нашей схемы:

npx prisma migrate dev --name init
# --name init — название миграции (можно любое)
# Команда:
#  1. Создаст файл migrations/xxxx_init/migration.sql с SQL-командами
#  2. Выполнит SQL в базе данных — создаст таблицы
#  3. Сгенерирует TypeScript-клиент Prisma

# Проверить — открыть Prisma Studio (визуальный просмотр БД):
npx prisma studio
# Откроется браузер с таблицами, можно добавить тестовые данные
💡 Prisma Studio — это как pgAdmin, но проще. Открывается по адресу http://localhost:5555. Там можно просматривать и редактировать данные без SQL.
Шаг 2

🖥️ Пишем Express-сервер

5

Создать точку входа сервера

Создайте файл src/index.ts:

backend/src/index.ts
import express from 'express'         // импортируем Express
import cors    from 'cors'            // CORS — разрешаем запросы с других доменов
import dotenv  from 'dotenv'          // читаем .env файл
import { PrismaClient } from '@prisma/client'

dotenv.config() // загрузить переменные из .env в process.env

export const prisma = new PrismaClient() // единственный экземпляр на весь сервер

const app = express()

// Middleware — функции, которые обрабатывают каждый запрос:
app.use(cors({
  origin: 'http://localhost:5173' // разрешаем только запросы от нашего React
}))
app.use(express.json()) // автоматически парсить JSON из тела запроса

// Импортируем роуты (создадим в следующем шаге)
import productsRouter from './routes/products'
import cartRouter     from './routes/cart'

app.use('/api/products', productsRouter) // все роуты товаров начинаются с /api/products
app.use('/api/cart',     cartRouter)     // все роуты корзины начинаются с /api/cart

// Запустить сервер на порту из .env (или 3001 по умолчанию)
const PORT = process.env.PORT || 3001
app.listen(PORT, () => {
  console.log(`✅ Сервер запущен: http://localhost:${PORT}`)
})
6

Создать роуты для товаров

Создайте файл src/routes/products.ts:

backend/src/routes/products.ts
import { Router } from 'express'
import { prisma } from '../index'

const router = Router() // создаём отдельный роутер для товаров

// GET /api/products — получить все товары
router.get('/', async (req, res) => {
  // async — потому что запрос к БД занимает время (асинхронная операция)
  try {
    const products = await prisma.product.findMany()
    // prisma.product.findMany() = SELECT * FROM products
    // await — ждём пока PostgreSQL вернёт результат

    res.json(products) // отправляем массив товаров клиенту в формате JSON
  } catch (error) {
    res.status(500).json({ error: 'Ошибка сервера' })
    // 500 = Internal Server Error — что-то пошло не так
  }
})

// GET /api/products/:id — получить один товар по ID
router.get('/:id', async (req, res) => {
  const id = Number(req.params.id) // req.params.id — строка из URL, конвертируем в число

  const product = await prisma.product.findUnique({
    where: { id } // WHERE id = :id
  })

  if (!product) {
    return res.status(404).json({ error: 'Товар не найден' })
    // 404 = Not Found
  }

  res.json(product)
})

export default router
7

Создать роуты для корзины

Создайте файл src/routes/cart.ts:

backend/src/routes/cart.ts
import { Router } from 'express'
import { prisma } from '../index'

const router = Router()

// POST /api/cart — добавить товар в корзину
router.post('/', async (req, res) => {
  const { productId } = req.body // получаем productId из тела запроса
  const userId = 1 // пока используем фиктивного пользователя (авторизацию добавим в лаб.5)

  const item = await prisma.cartItem.upsert({
    // upsert = UPDATE если существует, INSERT если нет
    where: {
      userId_productId: { userId, productId } // ищем по составному уникальному ключу
    },
    update: {
      qty: { increment: 1 } // если товар уже в корзине — увеличиваем qty на 1
    },
    create: {
      userId, productId, qty: 1 // если нет — создаём новую запись
    },
    include: { product: true } // в ответе включить связанный товар (JOIN)
  })

  res.status(201).json(item) // 201 = Created
})

// GET /api/cart — получить всю корзину пользователя
router.get('/', async (req, res) => {
  const userId = 1

  const items = await prisma.cartItem.findMany({
    where: { userId },     // WHERE user_id = 1
    include: { product: true } // JOIN с таблицей products
  })

  // Считаем сумму корзины на сервере
  const total = items.reduce((sum, i) => sum + i.product.price * i.qty, 0)

  res.json({ items, total })
})

// DELETE /api/cart/:id — удалить позицию из корзины
router.delete('/:id', async (req, res) => {
  const id = Number(req.params.id)
  await prisma.cartItem.delete({ where: { id } })
  res.json({ success: true })
})

export default router

import { PrismaClient } from '@prisma/client'
const prisma = new PrismaClient()

async function main() {
  await prisma.product.createMany({
    data: [
      { name: 'Кроссовки Nike',    price: 4990, oldPrice: 6500, stock: 15 },
      { name: 'Футболка Adidas',   price: 1290,               stock: 42 },

  })

}

main().finally(() => prisma.$disconnect())

# Запустить seed:
npx ts-node seed.ts

Проверить API в браузере и Postman

В браузере (только GET):

Откройте URL в адресной строке:

http://localhost:3001/api/products
http://localhost:3001/api/products/1

В Postman (все методы):

POST http://localhost:3001/api/cart
Body (JSON):
{
  "productId": 1
}
📱 Postman — бесплатная программа для тестирования API. Скачать: postman.com
Позволяет отправлять POST/PATCH/DELETE запросы с телом (body) — то, что браузер не умеет из адресной строки.

Ожидаемые ответы:

// GET /api/products — массив товаров:
[
  { "id": 1, "name": "Кроссовки Nike", "price": 4990, "stock": 15 },
  { "id": 2, "name": "Футболка Adidas", "price": 1290, "stock": 42 }
]

// POST /api/cart — добавленная позиция корзины:
{
  "id": 1,
  "qty": 1,
  "productId": 1,
  "userId": 1,
  "product": { "id": 1, "name": "Кроссовки Nike", ... }
}
