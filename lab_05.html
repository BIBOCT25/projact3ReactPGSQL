🔐 Как работает JWT-авторизация

JWT (JSON Web Token) — это строка-«билет», которую сервер выдаёт после входа. Клиент хранит этот билет и при каждом запросе прикладывает его в заголовке. Сервер проверяет подпись билета и знает: кто этот пользователь.
1. Клиент
POST /api/auth/login
email + password
→
2. Сервер
bcrypt.compare()
выдаёт токен
→
3. Клиент
сохраняет в
localStorage
4. Следующий запрос
GET /api/cart
Authorization: Bearer <token>
→
5. Middleware
jwt.verify()
достаёт userId
→
6. Роут
req.userId = 42
отвечает данными

Структура токена

// Токен состоит из трёх частей, разделённых точкой:
eyJhbGciOiJIUzI1NiJ9     ← Header: алгоритм подписи
.eyJ1c2VySWQiOjQyfQ      ← Payload: данные (userId, exp)
.SflKxwRJSMeKKF2QT4fw    ← Signature: подпись (секретный ключ)

// Payload после декодирования (base64):
{ "userId": 42, "exp": 1746300000 }
// exp — время истечения (unix timestamp)
⚠️ JWT — не шифрование, а подпись!
Payload токена можно декодировать без ключа (он в base64). Никогда не кладите в токен пароли или секретные данные. Подпись лишь гарантирует что токен не подделан — без секретного ключа не получится создать валидный токен.

🛡️ Middleware проверки токена

3

Создать src/middleware/auth.ts

Этот файл будет подключаться к любому роуту, который требует авторизации.

backend/src/middleware/auth.ts
import { Request, Response, NextFunction } from 'express'
import jwt from 'jsonwebtoken'

// Расширяем тип Request — добавляем поле userId
declare global {
  namespace Express {
    interface Request {
      userId?: number
    }
  }
}

export function requireAuth(req: Request, res: Response, next: NextFunction) {
  // Заголовок Authorization: "Bearer eyJ..."
  const header = req.headers.authorization
  const token  = header?.split(' ')[1]   // берём часть после "Bearer "

  if (!token) {
    return res.status(401).json({ error: 'Токен не передан' })
    // 401 Unauthorized — запрос без авторизации
  }

  try {
    const payload = jwt.verify(token, process.env.JWT_SECRET!) as { userId: number }
    // jwt.verify: проверяет подпись и срок действия токена
    // Если токен просрочен или подделан — бросает исключение

    req.userId = payload.userId  // кладём userId в request — роут его прочитает
    next()                       // передать управление следующему обработчику
  } catch {
    res.status(401).json({ error: 'Токен недействителен или истёк' })
  }
}
💡 next() — ключевое слово middleware.
Express обрабатывает запрос через цепочку функций. next() говорит «я закончил, передай следующему». Если не вызвать next() и не отправить ответ — запрос зависнет.

🔑 Роуты регистрации и входа

4

Создать src/routes/auth.ts

backend/src/routes/auth.ts
import { Router }  from 'express'
import bcrypt      from 'bcryptjs'
import jwt         from 'jsonwebtoken'
import { prisma }  from '../index'

const router = Router()

// POST /api/auth/register — регистрация
router.post('/register', async (req, res) => {
  const { email, password } = req.body

  // Базовая валидация
  if (!email || !password) {
    return res.status(400).json({ error: 'Укажите email и пароль' })
  }
  if (password.length < 6) {
    return res.status(400).json({ error: 'Пароль минимум 6 символов' })
  }

  // Проверить что email не занят
  const exists = await prisma.user.findUnique({ where: { email } })
  if (exists) {
    return res.status(409).json({ error: 'Email уже занят' })
    // 409 Conflict
  }

  // Хешировать пароль (10 — сложность, больше = медленнее, но безопаснее)
  const passwordHash = await bcrypt.hash(password, 10)
  // bcrypt.hash превращает "mypassword" → "$2a$10$..."
  // Из хеша нельзя получить оригинал — это одностороннее преобразование

  const user = await prisma.user.create({
    data: { email, passwordHash }
  })

  // Выдать токен сразу после регистрации
  const token = signToken(user.id)
  res.status(201).json({ token, userId: user.id })
})

// POST /api/auth/login — вход
router.post('/login', async (req, res) => {
  const { email, password } = req.body

  const user = await prisma.user.findUnique({ where: { email } })
  if (!user) {
    return res.status(401).json({ error: 'Неверный email или пароль' })
    // Намеренно одинаковое сообщение — не раскрываем существует ли email
  }

  // Сравнить введённый пароль с хешем в БД
  const match = await bcrypt.compare(password, user.passwordHash)
  // bcrypt.compare хеширует password и сравнивает с сохранённым хешем
  if (!match) {
    return res.status(401).json({ error: 'Неверный email или пароль' })
  }

  const token = signToken(user.id)
  res.json({ token, userId: user.id })
})

// Вспомогательная функция создания токена
function signToken(userId: number): string {
  return jwt.sign(
    { userId },                   // payload — что зашиваем в токен
    process.env.JWT_SECRET!,      // секретный ключ из .env
    { expiresIn: '7d' }           // токен действует 7 дней
  )
}

export default router

⚛️ Frontend: Context и хук авторизации

6

Создать AuthContext — глобальное состояние

Context позволяет передавать данные (токен, userId) любому компоненту без «прокидывания через props».

frontend/src/context/AuthContext.tsx
import { createContext, useContext, useState, ReactNode } from 'react'

interface AuthCtx {
  token:   string | null
  userId:  number | null
  login:   (token: string, userId: number) => void
  logout:  () => void
}

// Создаём контекст с начальным значением null
const AuthContext = createContext<AuthCtx | null>(null)

// Provider — оборачивает всё приложение, предоставляет данные вниз
export function AuthProvider({ children }: { children: ReactNode }) {
  // При старте читаем токен из localStorage (восстановление сессии)
  const [token,  setToken]  = useState<string | null>(
    localStorage.getItem('token')
  )
  const [userId, setUserId] = useState<number | null>(
    Number(localStorage.getItem('userId')) || null
  )

  const login = (t: string, id: number) => {
    localStorage.setItem('token',  t)
    localStorage.setItem('userId', String(id))
    setToken(t)
    setUserId(id)
  }

  const logout = () => {
    localStorage.removeItem('token')
    localStorage.removeItem('userId')
    setToken(null)
    setUserId(null)
  }

  return (
    <AuthContext.Provider value={{ token, userId, login, logout }}>
      {children}  {'// все дочерние компоненты получат доступ к контексту'}
    </AuthContext.Provider>
  )
}

// Удобный хук для использования контекста
export function useAuth() {
  const ctx = useContext(AuthContext)
  if (!ctx) throw new Error('useAuth вне AuthProvider')
  return ctx
}

Форма входа/регистрации LoginPage.tsx

Вход в аккаунт

frontend/src/pages/LoginPage.tsx
import { useState }        from 'react'
import { useNavigate }     from 'react-router-dom'
import { useAuth }        from '../context/AuthContext'

export function LoginPage() {
  const [email,    setEmail]    = useState('')
  const [password, setPassword] = useState('')
  const [isReg,    setIsReg]    = useState(false)  // false = вход, true = регистрация
  const [error,    setError]    = useState('')
  const [loading,  setLoading]  = useState(false)

  const { login }   = useAuth()
  const navigate    = useNavigate()

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()  // не перезагружать страницу при сабмите формы
    setError('')
    setLoading(true)

    try {
      const url = isReg ? '/api/auth/register' : '/api/auth/login'
      const res = await fetch(url, {
        method:  'POST',
        headers: { 'Content-Type': 'application/json' },
        body:    JSON.stringify({ email, password })
      })

      if (!res.ok) {
        const data = await res.json()
        throw new Error(data.error)
      }

      const { token, userId } = await res.json()
      login(token, userId)    // сохранить в Context + localStorage
      navigate('/')           // перенаправить на каталог
    } catch (err: any) {
      setError(err.message)
    } finally {
      setLoading(false)
    }
  }

  return (
    <form className="auth-form" onSubmit={handleSubmit}>
      <h2>{isReg ? 'Регистрация' : 'Вход'}</h2>

      {error && <p className="auth-error">{error}</p>}

      <input type="email"    placeholder="Email"
             value={email}    onChange={e => setEmail(e.target.value)} required />

      <input type="password" placeholder="Пароль"
             value={password} onChange={e => setPassword(e.target.value)} required />

      <button type="submit" disabled={loading}>
        {loading ? 'Загрузка…' : isReg ? 'Зарегистрироваться' : 'Войти'}
      </button>

      <p>
        {isReg ? 'Уже есть аккаунт?' : 'Нет аккаунта?'}
        <button type="button" onClick={() => setIsReg(!isReg)}>
          {isReg ? 'Войти' : 'Зарегистрироваться'}
        </button>
      </p>
    </form>
  )
}

import { Navigate } from 'react-router-dom'
import { useAuth }  from '../context/AuthContext'

export function ProtectedRoute({ children }: { children: JSX.Element }) {
  const { token } = useAuth()
  if (!token) return <Navigate to="/login" replace />
  // replace — не добавлять /cart в историю браузера
  return children
}

Добавить токен во все fetch-запросы

Обновить api/cart.ts — передавать токен в заголовке каждого запроса.

frontend/src/api/cart.ts — обновить
// Вспомогательная функция — заголовки с токеном
function authHeaders() {
  const token = localStorage.getItem('token')
  return {
    'Content-Type': 'application/json',
    ...(token ? { 'Authorization': `Bearer ${token}` } : {})
    // spread-оператор: добавить Authorization только если токен есть
  }
}

export async function fetchCart() {
  const res = await fetch('/api/cart', { headers: authHeaders() })
  if (res.status === 401) throw new Error('Необходима авторизация')
  return res.json()
}

🔒 Важно о безопасности:
  • Никогда не храните пароли в открытом виде — только bcrypt-хеш
  • JWT_SECRET должен быть длинным случайным ключом (минимум 32 символа)
  • localStorage уязвим к XSS — для production лучше httpOnly cookie
  • Всегда проверяйте токен на сервере — клиенту нельзя доверять
